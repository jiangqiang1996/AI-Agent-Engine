# AE Brainstorm

目标：
- 澄清需求边界
- 产出 `docs/brainstorms/*-requirements.md`
- 在文档生成或更新后，转入 `ae:document-review`

执行原则：
- 以中文与用户协作
- 优先形成可规划的 requirements，而不是直接进入实现细节
- 仅在确有需要时追问，问题一次只问一个
- 若已存在相关 requirements 文档，优先续写而不是重复创建
