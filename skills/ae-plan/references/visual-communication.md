# 计划文档中的视觉传达

此指南覆盖计划结构层面的视觉辅助——依赖图、交互图和比较表。Section 3.4 的高层技术设计图表是不同的关注点。

视觉辅助取决于内容模式，而非计划深度分类。

**何时包含：**

| 计划描述了... | 视觉辅助 | 放置位置 |
|---|---|---|
| 4+ 个非线性依赖的实现单元 | Mermaid 依赖图 | 实现单元标题附近 |
| 系统范围影响中 3+ 个交互面 | Mermaid 交互图 | 系统范围影响部分内 |
| 概览涉及 3+ 个行为模式 | Markdown 比较表 | 概览或问题框架内 |
| 关键技术决策有 3+ 个交互决策 | Markdown 比较表 | 相关部分内 |

**何时跳过：**
- 3 个或更少线性单元
- 散文已经清晰传达了关系
- 视觉重复高层技术设计已展示的内容
- 视觉描述代码级细节

**格式选择：**
- **Mermaid**（默认）用于依赖图和交互图——使用 `TB` 方向
- **ASCII/框线图**用于需要丰富框内内容的注释流程
- **Markdown 表格**用于比较
- 保持图表成比例
- 散文是权威的

生成视觉辅助后，验证它准确表示了计划部分。
