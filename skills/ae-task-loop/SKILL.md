# 迭代任务循环

目标驱动的迭代执行器。每轮"执行→验证→判定"，直到目标达成或策略穷尽时退出。与 `ae:work`（计划驱动）互补。

**适用场景**（`ae:work` 不擅长或无法覆盖的）：

- 探索性调试："修复这个 flaky test，不确定根因在哪"
- 环境搭建："配置这个项目的 CI/CD 直到所有检查通过"
- 遗留代码修复："消除这个模块的所有 TypeScript 编译错误"

**不适用场景**：

- 有明确结构化计划的实现 → 使用 `ae:work`
- 需求尚不明确、需要探索方案 → 使用 `ae:brainstorm`
- 代码变更后的质量审查 → 使用 `ae:review`

## 输入

<goal> #$ARGUMENTS </goal>

如果 `<goal>` 为空，询问用户："你想完成什么目标？"

## 铁律

- **T1. 循环体禁言令**：从进入第 1 轮循环起，整个循环体（执行、验证、判定）严格禁止向用户提问。允许提问的退出条件仅限：(a) 达到迭代上限后；(b) 遇到不可恢复错误后。
- **T2. 执行技能禁言穿透**：执行子代理必须跳过被调用技能中所有询问用户的环节，使用第 0 步已确认的答案或合理默认值继续。

## 已知技能名列表

> 以下为可识别的技能名前缀。更新时同步 `ae-catalog.ts`。

ae:work, ae:review, ae:brainstorm, ae:plan, ae:plan-review, ae:document-review, ae:ideate, ae:sql, ae:test-browser, ae:setup, ae:frontend-design, ae:save-rules, ae:handoff, ae:help, ae:update, ae:lfg

## 流程

```
第 0 步：理解 & 确认（仅一次，不计入轮次）
    ↓
循环体：执行（子代理）→ 验证（子代理）→ 判定（当前会话）
    ↓
退出处理
```

**轮次计数**：第 0 步不计入轮次。每启动一次执行子代理，轮次 +1。默认最大 30 轮。

**退出语义**（内联，完成/瓶颈/上限/不可恢复异常/基础设施异常）：
- 所有完成条件满足 → DONE，结束
- 达到最大轮次 → 退出处理，允许提问
- 连续两轮无进展 → 瓶颈退出
- 执行子代理标记 UNRECOVERABLE → 快速退出，允许提问
- 连续两轮执行失败且无有效进展 → 启发式退出，允许提问
- 连续两轮验证子代理失败 → 基础设施异常退出

### 第 0 步：理解与确认

参考：@./references/phase-zero.md

1. 解析目标，识别执行技能（第一个 token 匹配技能名列表则视为技能名）
2. 分析执行技能交互需求（读取 SKILL.md，识别交互节点，分为可默认/不可默认）
3. 探测项目工具链（仅代码任务）
4. 分析潜在隐患和风险点
5. 推导完成条件（3-8 条，附带 ID）
6. 统一展示所有待确认项，等待用户确认后才进入循环

### 循环体

参考：@./references/loop-body.md

- **执行**：子代理完成，可插拔技能。详见 loop-body.md 中的子代理 prompt 模板和禁言令声明
- **验证**：子代理逐项对照完成条件验证。工具链探测结果注入验证子代理 prompt
- **判定**：当前会话直接执行，对照退出语义做判断

### 退出处理

参考：@./references/exit-handling.md

- 迭代上限：输出结论 + 用户选择（继续/结束/其他）
- 不可恢复错误：双层判定（快速通道 + 启发式兜底），输出结论 + 用户选择
- 基础设施异常：直接输出异常报告
- 全部完成：直接输出 DONE

## 执行约束

- 破坏性操作依赖 opencode 权限系统控制
- 历史失败摘要每轮压缩为最近 3 轮的关键信息
