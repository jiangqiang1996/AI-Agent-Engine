# 迭代任务循环

目标驱动的迭代执行器。每轮"执行→验证→判定"，直到目标达成或策略穷尽时退出。与 `ae:work`（计划驱动）互补。

**适用场景：** 探索性调试、环境搭建、遗留代码修复

**不适用场景：** 有明确计划 → `ae:work`；需求不明确 → `ae:brainstorm`；代码审查 → `ae:review`

## 输入

<goal> #$ARGUMENTS </goal>

如果 `<goal>` 为空，询问用户："你想完成什么目标？"

## 铁律

- **T1. 循环体禁言令**：从进入第 1 轮循环起，整个循环体严格禁止向用户提问。允许提问的退出条件仅限：(a) 达到迭代上限后；(b) 遇到不可恢复错误后。
- **T2. 执行技能禁言穿透**：执行子代理必须跳过所有询问用户的环节，使用已确认的答案或合理默认值。

## 流程

```
第 0 步：理解 & 确认（仅一次，不计入轮次）
    ↓
循环体：执行（子代理）→ 验证（子代理）→ 判定（当前会话）
    ↓
退出处理
```

**轮次计数：** 第 0 步不计入。每启动一次执行子代理 +1。默认最大 30 轮。

**退出语义：**
- 所有完成条件满足 → DONE
- 达到最大轮次 → 退出处理，允许提问
- 连续两轮无进展 → 瓶颈退出
- 执行子代理标记 UNRECOVERABLE → 快速退出
- 连续两轮执行失败 → 启发式退出
- 连续两轮验证子代理失败 → 基础设施异常退出

### 第 0 步：理解与确认

参考：@./references/phase-zero.md

1. 解析目标，识别执行技能
2. 分析执行技能交互需求
3. 探测项目工具链（仅代码任务）
4. 分析潜在隐患
5. 推导完成条件（3-8 条，附带 ID）
6. 统一展示所有待确认项，等待确认

### 循环体

参考：@./references/loop-body.md

- **执行**：子代理完成，可插拔技能
- **验证**：子代理逐项对照完成条件验证
- **判定**：当前会话直接执行，对照退出语义做判断

### 退出处理

参考：@./references/exit-handling.md

- 迭代上限/不可恢复错误：输出结论 + 用户选择
- 基础设施异常：直接输出异常报告
- 全部完成：直接输出 DONE

## 执行约束

- 破坏性操作依赖 opencode 权限系统控制
- 历史失败摘要每轮压缩为最近 3 轮的关键信息
