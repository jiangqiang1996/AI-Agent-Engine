# 退出处理

## 迭代上限退出

达到最大轮次时：

```markdown
### 循环结束

**退出原因**：轮次上限（N/30）

**状态摘要**：
- 已完成项：[条件 ID 列表]
- 未完成项：[条件 ID 列表]

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

**允许提问**：继续迭代 / 结束执行 / 其他

## 不可恢复错误退出

### 双层判定

- **快速通道**：子代理标记 "UNRECOVERABLE" 时立即采纳
- **启发式兜底**：连续两轮相同根因失败且无进展

### 退出流程

输出结论（包含错误信息、状态摘要、轮次、变更文件），然后允许提问。

## 基础设施异常退出

连续两轮验证子代理失败时直接输出异常报告，不进入用户选择流程。

```markdown
### 循环结束

**退出原因**：基础设施异常（验证子代理连续失败）

**异常信息**：[失败描述]

**状态摘要**：
- 已完成项：[条件 ID 列表]
- 未完成项：[条件 ID 列表]

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

## 全部完成退出

```markdown
### 循环结束

**退出原因**：完成

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

标记 DONE，结束。无需额外确认。

## 用户选择菜单（迭代上限和不可恢复错误共用）

```
循环已暂停。请选择后续方案：
1. 继续迭代 — 指定增加的轮次数（如 +10）
2. 结束执行 — 保留当前状态
3. 其他 — 描述你想要的方案
```
