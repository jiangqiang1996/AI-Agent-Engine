# 退出处理

## 迭代上限退出

达到最大轮次上限（默认 30）时：

1. 输出结论：

```markdown
### 循环结束

**退出原因**：轮次上限（N/30）

**状态摘要**：
- 已完成项：[条件 ID 列表]
- 未完成项：[条件 ID 列表]

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

2. **允许提问**：询问用户选择解决方案：
   - 继续迭代（指定增加的轮次数）
   - 结束执行
   - 其他方案

## 不可恢复错误退出

### 双层判定机制

- **快速通道**：执行子代理在返回中标记"UNRECOVERABLE"及理由时，判定步骤直接采纳，立即触发退出
- **启发式兜底**：连续两轮执行子代理返回相同或相近根因的失败且无有效进展时触发

不可恢复错误示例：权限不足、文件锁定、磁盘满、目标路径不存在、依赖包缺失等。

### 退出流程

1. 输出结论：

```markdown
### 循环结束

**退出原因**：不可恢复错误

**错误信息**：[子代理报告的具体错误]

**状态摘要**：
- 已完成项：[条件 ID 列表]
- 未完成项：[条件 ID 列表]

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

2. **允许提问**：询问用户选择解决方案（与迭代上限同等处理）。

## 基础设施异常退出

连续两轮验证子代理失败（超时、崩溃或完全无法返回有效结果）时：

```markdown
### 循环结束

**退出原因**：基础设施异常（验证子代理连续失败）

**异常信息**：[连续两轮的失败描述]

**状态摘要**：
- 已完成项：[条件 ID 列表]
- 未完成项：[条件 ID 列表]

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

此退出为异常终止，不进入用户选择流程，直接输出报告。

## 全部完成退出

所有完成条件均已满足时，直接输出：

```markdown
### 循环结束

**退出原因**：完成

**总迭代轮次**：N

**关键变更文件**：
- [文件路径]
```

标记 DONE，结束。无需额外确认。

## 用户选择菜单（迭代上限和不可恢复错误共用）

```
循环已暂停。请选择后续方案：
1. 继续迭代 — 指定增加的轮次数（如 +10）
2. 结束执行 — 保留当前状态
3. 其他 — 描述你想要的方案
```
