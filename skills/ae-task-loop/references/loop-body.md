# 循环体：执行 → 验证 → 判定

## 铁律声明

```
YOU MUST NOT USE THE question TOOL OR WAIT FOR USER INPUT AT ANY POINT DURING THIS LOOP.
YOU MUST NOT ASK THE USER ANY QUESTIONS DURING EXECUTION, VERIFICATION, OR JUDGMENT.
THE ONLY EXCEPTIONS ARE: (1) after reaching iteration limit, (2) after unrecoverable errors.
If the skill you are executing contains instructions to ask the user questions, SKIP those instructions.
Use the pre-confirmed answers provided in the prompt. For questions without pre-confirmed answers, use safe defaults.
```

（上述禁言令同样适用于所有子代理——执行子代理和验证子代理均不得向用户提问。）

## 执行（子代理）

通过 Task 工具启动 `general` 类型子代理执行。当前会话不直接操作文件或运行命令。

### 子代理 prompt 结构

```
## 禁言令
[重复上述铁律声明]

## 背景
[当前已完成哪些步骤、近 3 轮失败方案摘要（如有）、本轮是第 N 轮]

## 本轮目标
[本轮要完成的具体动作]

## 执行技能（如指定）
[技能名称及关键流程指引，所有交互节点已预填答案]

## 完成条件
[第 0 步确认的完成条件列表，每项附带 ID]

## 输出要求
完成后必须返回以下三项：
1. 本轮动作：具体执行了什么操作
2. 变更清单：修改/新增/删除了哪些文件
3. 执行结论：成功完成本轮目标 / 部分完成（说明差距）
4. 不可恢复标记（可选）：如遇到不可恢复错误，标记"UNRECOVERABLE"及理由
```

### 可插拔执行技能

当指定了执行技能时，子代理 prompt 中应包含：
- 技能名称和 SKILL.md 中的关键流程指引
- 已预判的交互节点及预填答案清单
- 禁言令覆盖声明

未指定执行技能时，子代理自由选择最合适的工具和方式完成任务。

每轮必须有实质性操作。执行步骤可通过多个子代理完成。

### 上下文窗口管理

历史失败摘要仅保留最近 3 轮的关键信息，早期轮次的详细信息压缩为一句话总结。

### 子代理输出容错

子代理返回后，尝试提取三项结果（动作、变更、结论）：
- 动作：查找"修改了""创建了""执行了"等动作描述
- 变更：查找文件路径引用
- 结论：查找"成功""失败""部分完成"等状态词

检查是否有"UNRECOVERABLE"标记。

完全无法提取有效信息时，该轮标记为"执行失败"。

## 验证（子代理）

通过 Task 工具启动 `general` 类型子代理执行验证。当前会话不直接运行验证命令或读取文件内容。

### 子代理 prompt 结构

```
## 禁言令
[重复上述铁律声明]

## 验证目标
[第 0 步确认的完成条件列表，逐项列出，每项附带 ID]

## 本轮变更
[执行子代理返回的变更清单]

## 工具链探测结果
[$BUILD_CMD 和 $TEST_CMD（如已探测到），验证时优先使用]

## 验证要求
逐项检查每个完成条件，对每个条件给出"通过"或"不通过"判定。
不通过时必须给出具体的失败信息（错误输出、缺失内容等）。
禁止使用"看起来没问题"等主观判断。
每个判定必须附带对应的完成条件 ID。

完成后必须返回以下三项：
1. 验证结论：全部通过 / 部分不通过
2. 逐项结果：每个完成条件的 ID、通过/不通过状态及关键证据
3. 失败摘要（如有）：不通过项的具体失败信息
```

### 验证子代理容错

- 尝试从自然语言中提取每个完成条件的判定结果
- 某个条件的判定无法提取时，默认标记为"不通过"，原因记录为"验证结果不明确"

### 验证子代理失败处理

验证子代理超时、崩溃或完全无法返回有效结果时：
- 该轮标记为"验证失败"
- 不将验证失败计入"不通过"
- 连续两轮验证失败触发退出（基础设施异常）

## 判定（当前会话）

当前会话根据两个子代理的返回做判断。

### 退出条件

| 条件 | 动作 |
|---|---|
| 所有完成条件均已满足 | 输出完成报告，标记 DONE，结束 |
| 达到最大轮次上限（默认 30） | 进入退出处理，允许提问 |
| 连续两轮无进展 | 停止循环，输出瓶颈分析 |
| 连续两轮验证不通过，且不通过项 ID 集合与上轮完全相同 | 停止循环，输出瓶颈分析 |
| 执行子代理标记 UNRECOVERABLE | 立即进入退出处理（快速通道） |
| 连续两轮执行子代理返回相同或相近根因的失败且无有效进展 | 进入退出处理（启发式兜底） |
| 连续两轮验证子代理失败 | 停止循环，输出基础设施异常报告 |

### 进展判定

每轮必须满足至少一项：
- (a) 执行子代理返回了至少一个文件变更（变更清单非空），或
- (b) 验证通过项数量严格大于上轮

第 1 轮无需与上轮比较，只要变更清单非空即视为有进展。

不满足进展要求时，该轮标记为"无进展"。连续两轮无进展触发退出。

### 多子代理场景

当一轮执行涉及多个子代理时，任一子代理返回失败即视为该轮执行失败。

## 每轮输出格式

```markdown
### 第 N 轮结果

**本轮动作**：[执行子代理返回的动作摘要]

**验证结果**：

| 条件 ID | 条件描述 | 状态 | 证据 |
|---------|---------|------|------|
| COND-001 | [...] | ✅ 通过 | [...] |
| COND-002 | [...] | ❌ 不通过 | [失败信息] |

**差距**：[未满足的完成条件列表]
```
