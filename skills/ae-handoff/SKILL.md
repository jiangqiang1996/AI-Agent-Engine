# 会话交接技能 ae:handoff

## 功能说明
用户触发 `/ae-handoff [压缩等级]` 命令后，自动执行以下流程：
1. 获取当前会话完整消息历史
2. 智能提取四类核心信息：
   - 核心结论：会话达成的最终结论、问题解决方案
   - 已做决策：确定的技术选型、方案选择、优先级决策
   - 待办事项：未完成的任务清单、下一步计划
   - 项目上下文：项目基本信息、环境配置、参数约定
3. 敏感信息自动脱敏：密钥、密码、个人信息等自动替换为`***`
4. 支持压缩等级参数（1-5，默认1）：
   - 等级1：默认，最详细，保留最多内容，最大支持32000 tokens
   - 等级2：详细，最大支持16000 tokens
   - 等级3：中等压缩，最大支持8000 tokens
   - 等级4：简洁，最大支持4000 tokens
   - 等级5：最简洁，保留核心信息，最大支持2000 tokens
5. 创建独立新会话，不继承原会话历史
6. 将提取的上下文注入新会话系统提示词，确保新会话知晓前序结论
7. 若平台不支持自定义系统提示词，降级为将上下文作为新会话第一条不可编辑的消息
8. 自动导航终端到新会话窗口（类似执行 /new 的效果），用户无需手动切换
9. 返回新会话访问链接和完整的提取内容，不限制显示长度

## 执行流程
### 阶段 1：数据准备
1. 调用SDK API获取当前会话完整消息历史
2. 处理空会话场景，返回友好提示

### 阶段 2：信息提取与脱敏
1. 对会话内容先进行脱敏处理，再调用LLM提取四类核心信息
2. 自动检测并脱敏所有敏感信息：密钥、Token、密码、手机号、身份证号等
3. 处理超长会话：超过LLM上下文限制时优先保留最新消息，截断非关键聊天内容

### 阶段 3：新会话创建与交付
1. 调用SDK API创建全新独立会话
2. 优先将提取的核心上下文注入新会话系统提示词（system 字段）
3. 若系统提示词注入失败，降级为将上下文作为新会话第一条消息注入
4. 调用 TUI API 导航终端到新会话窗口（发送 tui.session.select 事件）
5. TUI 导航失败不影响交接结果，用户仍可手动切换
6. 返回新会话访问链接和提取摘要
