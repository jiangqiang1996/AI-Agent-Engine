# 会话交接工具 ae:handoff

## 功能说明
基于用户提供的结构化上下文创建全新独立会话，将上下文注入新会话，并自动跳转到新窗口。

1. 接收用户提供的结构化上下文（用户请求、目标、已完成工作、当前状态、待办任务、关键文件、重要决策、明确约束、续会注意事项）
2. 支持压缩等级参数（1-5，默认1），用于标注交接内容的详细程度偏好
3. 创建完全独立的新会话，不继承原会话历史记录
4. 将提取的结构化上下文注入新会话系统提示词，确保新会话掌握前序结论
5. 基于提取的核心信息生成精简标题（10-20字），新会话标题格式为「交接：[精简描述]」
6. 若平台不支持自定义系统提示词，自动降级为将上下文作为新会话第一条消息注入
7. 调用 TUI 接口跳转到新会话窗口（发送 tui.session.select 事件）
8. TUI 跳转失败不影响交接结果，可手动切换到新会话
9. 返回新会话访问链接和完整结构化提取摘要

## 执行流程

### 阶段 1：数据准备
用户通过工具参数提供结构化上下文信息，包括：
- 用户原始请求
- 任务目标
- 已完成工作
- 当前状态
- 待办任务
- 关键文件
- 重要决策
- 明确约束
- 续会注意事项
- 压缩等级（1-5，默认1）

### 阶段 2：新会话创建与交付
1. 基于提供的信息生成精简会话标题
2. 调用SDK接口创建全新独立会话
3. 优先将结构化上下文注入新会话系统提示词（system 字段）
4. 若系统提示词注入失败，自动降级为将上下文作为新会话第一条消息注入
5. 调用 TUI 接口跳转到新会话窗口
6. 返回新会话访问链接和完整结构化提取摘要

## 约束规则
- KEY FILES 部分最多展示10个文件，按重要性排序
- GOAL 部分保持精简，不超过两句话
- 用户请求和明确约束必须完整保留原文，不得修改或转述
- 所有文件路径使用工作区相对路径
- 避免包含不必要的实现细节（变量名、常量值等），核心关键信息除外
