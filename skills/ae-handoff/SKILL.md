# 会话交接工具 ae:handoff

## 功能说明
用户执行 `/ae-handoff [压缩等级]` 命令后，自动完成以下流程：
1. 自动收集多维度上下文信息：会话历史、待办进度、Git变更记录
2. 智能提取结构化核心信息，覆盖续会所需全部关键内容
3. 敏感信息自动脱敏：密钥、令牌、密码、个人隐私信息等自动替换为`***`
4. 支持压缩等级参数（1-5，默认1）：
   - 等级1：默认，最详细，保留最多内容，最大支持32000 tokens
   - 等级2：详细，最大支持16000 tokens
   - 等级3：中等压缩，最大支持8000 tokens
   - 等级4：简洁，最大支持4000 tokens
   - 等级5：最简洁，仅保留核心信息，最大支持2000 tokens
5. 创建完全独立的新会话，不继承原会话历史记录
6. 将提取的结构化上下文注入新会话系统提示词，确保新会话完全掌握前序结论
7. 基于提取的核心信息生成精简标题（10-20字），新会话标题格式为「交接：[精简描述]」
8. 若平台不支持自定义系统提示词，自动降级为将上下文作为新会话第一条不可编辑的系统消息
9. 自动跳转到新会话窗口（效果等同于执行 /new 命令），无需手动切换
10. 返回新会话访问链接和完整提取内容，无长度限制

## 执行流程
### 阶段 1：数据准备
1. 调用SDK接口获取当前会话完整消息历史
2. 调用 `todoread()` 工具获取当前任务进度
3. 执行 Bash 命令 `git diff --stat HEAD~10..HEAD` 获取最近10次提交的文件变更统计
4. 执行 Bash 命令 `git status --porcelain` 获取未提交的变更
5. 空会话场景自动返回友好提示

### 阶段 2：信息提取与脱敏
1. 所有收集到的内容先进行脱敏处理，再调用大模型提取结构化信息
2. 自动检测并脱敏所有敏感信息：密钥、令牌、密码、手机号、身份证号、个人隐私信息等
3. 结构化提取内容（按优先级排序）：
   - **用户请求（原样）**：用户的原始需求，完整保留原文，不做任何修改
   - **目标**：一句话描述当前任务的最终目标和下一步方向
   - **已完成工作**：已经完成的任务列表，包含相关文件路径和关键实现决策
   - **当前状态**：代码库当前状态、构建/测试状态、环境配置状态
   - **待办任务**：未完成的任务清单、下一步计划、阻塞问题（合并todoread的结果）
   - **关键文件**：最多10个关键文件路径及简要说明，优先包含Git变更和会话中提到的文件
   - **重要决策**：已确定的技术选型、方案选择、优先级决策、权衡考量
   - **明确约束**：用户明确要求的限制、项目规范、AGENTS.md中的约定（完整保留原文）
   - **续会注意事项**：需要留意的坑点、警告、相关文档引用
4. 超长会话处理：超过大模型上下文限制时优先保留最新消息和高优先级内容，裁剪非关键聊天内容
5. 根据用户指定的压缩等级调整内容详细程度，等级越高仅保留优先级最高的核心信息

### 阶段 3：上下文格式化
采用统一结构化格式输出上下文：
```
HANDOFF CONTEXT
===============

USER REQUESTS (AS-IS)
---------------------
- [用户原始请求，原样保留]

GOAL
----
[一句话描述任务目标和下一步方向]

WORK COMPLETED
--------------
- [已完成的任务，包含相关文件路径]
- [关键实现决策]

CURRENT STATE
-------------
- [代码库当前状态]
- [构建/测试状态]
- [环境配置信息]

PENDING TASKS
-------------
- [待完成的任务，合并todoread结果]
- [阻塞问题和已知风险]

KEY FILES
---------
- [path/to/file1] - [文件简要作用]
- [path/to/file2] - [文件简要作用]
(最多10个文件，按重要性排序)

IMPORTANT DECISIONS
-------------------
- [技术决策和选择原因]
- [约定的规范和模式]

EXPLICIT CONSTRAINTS
--------------------
- [用户明确要求的限制]
- [项目规范约定]
- 无则填写：None

CONTEXT FOR CONTINUATION
------------------------
- [需要注意的坑点和警告]
- [相关文档和参考链接]
```

### 阶段 4：新会话创建与交付
1. 基于提取的核心信息生成精简会话标题，优先使用核心结论/主要任务作为标题内容，长度控制在10-20字
2. 调用SDK接口创建全新独立会话，传入生成的标题「交接：[精简描述]」
3. 优先将提取的结构化上下文注入新会话系统提示词（system 字段）
4. 若系统提示词注入失败，自动降级为将上下文作为新会话第一条消息注入
5. 调用 TUI 接口跳转到新会话窗口（发送 tui.session.select 事件）
6. TUI 跳转失败不影响交接结果，可手动切换到新会话
7. 返回新会话访问链接和完整结构化提取摘要

## 约束规则
- 禁止包含任何敏感信息（API密钥、凭证、密码等）
- KEY FILES 部分最多展示10个文件，按重要性排序
- GOAL 部分保持精简，不超过两句话
- 用户请求和明确约束必须完整保留原文，不得修改或转述
- 所有文件路径使用工作区相对路径
- 避免包含不必要的实现细节（变量名、常量值等），核心关键信息除外
