# Diff 范围规则

适用于每位审查者的范围规则。

## 范围发现

按以下优先级确定 diff：

1. **用户指定范围** — 调用方传入了 `BASE:`、`FILES:` 或 `DIFF:`
2. **工作副本变更** — 未暂存或已暂存的变更
3. **未推送提交与基准分支对比**

## 发现分类层级

### 主要（直接变更的代码）

diff 中新增或修改的行。使用完全置信度。

### 次要（紧邻的周围代码）

与变更行处于同一函数或代码块中的未变更代码。如果变更引入的 bug 只有通过阅读上下文才能发现，则报告。

### 预存（与本次 diff 无关）

diff 未触及且不与之交互的未变更代码。标记 `pre_existing: true`，不计入审查结论。

**规则：** 如果在不含该 diff 时也会标记同一问题，则它是预存的。如果 diff 使问题变得新相关，则它是次要的。
