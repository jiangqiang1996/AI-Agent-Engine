# Git 工作流规范

## 分支策略

### 分支命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 功能 | `feat/<简短描述>` | `feat/add-agent-tool` |
| 修复 | `fix/<简短描述>` | `fix/tool-execution-error` |
| 重构 | `refactor/<简短描述>` | `refactor/hook-layer` |
| 文档 | `docs/<简短描述>` | `docs/api-reference` |
| 测试 | `test/<简短描述>` | `test/tool-integration` |

### 分支流程

1. 从 `main` 创建功能分支
2. 开发完成后创建 Pull Request
3. 通过代码审查后合并到 `main`
4. 删除已合并的功能分支

## 提交规范

### 提交信息格式

```
<类型>(<范围>): <简短描述>

<详细描述>（可选）
```

### 类型标识

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 Bug |
| `refactor` | 重构（不改变外部行为） |
| `docs` | 文档变更 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |
| `perf` | 性能优化 |

### 示例

```
feat(tools): 添加文件搜索工具

支持按文件名和内容模式搜索项目文件，
返回匹配文件的路径和上下文片段。
```

```
fix(hooks): 修复消息转换中的空值处理

当消息内容为空时，跳过转换而非抛出异常。
```

### 提交粒度

- 每次提交只做一件事
- 提交内容必须可独立运行
- 禁止提交未完成的代码（WIP）
- 禁止提交调试代码（console.log 等）

## 禁止事项

- 禁止强制推送到 `main` 分支
- 禁止提交敏感信息（密钥、Token、密码等）
- 禁止提交 `node_modules`、`.env` 等文件
- 禁止在提交信息中包含无关内容
