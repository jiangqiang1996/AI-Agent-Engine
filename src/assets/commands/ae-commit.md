智能提交当前变更文件，自动生成符合项目规范的提交信息。

## 执行流程

### 1. 变更检测

运行 `git status` 和 `git diff --staged`，识别：
- **当前会话变更** — 优先提交本次会话中修改的文件
- **其他变更** — 会话无变更时，提交所有未暂存变更
- **无变更** — 提示用户无需提交，不创建空提交

### 2. 文件筛选

排除以下文件，禁止提交：
- 缓存文件、临时文件（如 `*.tmp`、`*.cache`）
- `node_modules/`、`.env`、敏感配置文件
- 密钥、Token、密码等敏感信息文件
- 构建产物（除非项目规范明确要求提交）
- 调试代码（含 `console.log` 等的文件需先清理）

### 3. 提交信息生成

提交信息必须遵循项目的 Git 提交规范（参见 `.opencode/rules/core/git-workflow.md`）：

格式：`<类型>(<范围>): <简短描述>`

类型标识：
- `feat` — 新功能
- `fix` — 修复 Bug
- `refactor` — 重构（不改变外部行为）
- `docs` — 文档变更
- `test` — 测试相关
- `chore` — 构建/工具变更
- `perf` — 性能优化

### 4. 提交粒度

遵循提交粒度规范：
- 每次提交只做一件事
- 提交内容必须可独立运行
- 禁止提交未完成代码（WIP）
- 禁止提交调试代码

当变更涉及多个独立关注点时，应拆分为多次提交。

### 5. 执行提交

```bash
git add <筛选后的文件>
git commit -m "<类型>(<范围>): <简短描述>"
```

### 6. 验证

提交后运行 `git status` 确认：
- 预期文件已提交
- 未预期文件未提交
- 工作目录状态正确

## 注意事项

- 提交信息必须使用中文
- 禁止推送到远程仓库（除非用户明确要求）
- 禁止强制推送到 `main` 分支
- 禁止修改 git config
- 禁止跳过 hooks（`--no-verify` 等）
- 禁止在提交信息或代码注释中出现"Created by Opencode"、"由 AI 代理生成"等表明代码由 AI 工具或代理生成的描述
- 当用户强制要求提交不合规内容时，遵从用户意愿但提醒规范
