# 会话交接

## 功能说明

基于用户提供的结构化上下文创建全新独立会话，将上下文注入新会话，并自动跳转到新窗口。

1. 接收用户提供的结构化上下文（用户请求、目标、已完成工作、当前状态、待办任务、关键文件、重要决策、明确约束、续会注意事项）
2. 支持压缩等级参数（1-5，默认 1），控制交接内容详细程度
3. 创建完全独立的新会话，不继承原会话历史记录
4. 将提取的结构化上下文注入新会话系统提示词
5. 基于核心信息生成精简标题（10-20 字），格式为「交接：[精简描述]」
6. 若平台不支持自定义系统提示词，自动降级为将上下文作为新会话第一条消息注入
7. 调用 TUI 接口跳转到新会话窗口
8. TUI 跳转失败不影响交接结果，可手动切换到新会话
9. 返回新会话访问链接和完整结构化提取摘要

## 执行流程

### 阶段 1：数据准备

用户通过工具参数提供结构化上下文信息：

- 用户原始请求
- 任务目标
- 已完成工作
- 当前状态
- 待办任务
- 关键文件
- 重要决策
- 明确约束
- 续会注意事项
- 压缩等级（1-5，默认 1）

#### 自动读取计划待办

当用户未显式提供待办任务时，自动调用 `ae-handoff` 工具内置的 `todoread` 能力读取计划文件中的待办任务列表：

1. 优先选择当前会话中使用过的计划文件
2. 多个计划文件时推荐最后使用的
3. 无会话相关计划时读取 `docs/plans/` 下最新的计划文件
4. 解析实现单元中的复选框任务
5. 支持通过 `file_path` 参数指定特定计划文件

`ae-handoff` 工具已内置此能力，无需单独调用其他工具。

### 阶段 2：新会话创建与交付

1. 基于提供的信息生成精简会话标题
2. 调用 SDK 接口创建全新独立会话
3. 优先将结构化上下文注入新会话系统提示词（system 字段）
4. 若系统提示词注入失败，自动降级为将上下文作为新会话第一条消息注入
5. 调用 TUI 接口跳转到新会话窗口
6. 返回新会话访问链接和完整结构化提取摘要

## 约束规则

- KEY FILES 部分最多展示 10 个文件，按重要性排序
- GOAL 部分保持精简，不超过两句话
- 用户请求和明确约束必须完整保留原文，不得修改或转述
- 所有文件路径使用工作区相对路径
- 避免包含不必要的实现细节（变量名、常量值等），核心关键信息除外
