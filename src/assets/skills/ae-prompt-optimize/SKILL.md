# 提示词优化

## 功能说明

将用户随意输入的提示词优化为结构化、高效的 AI 对话提示词，经用户确认后在新会话中自动提交执行。

1. 接收用户的原始提示词
2. 识别其中的命令、技能、工具、代理引用并确保保留
3. 对非引用部分进行结构化优化和上下文补充
4. 展示优化结果供用户确认或修改（auto 模式跳过）
5. 调用 `ae-prompt-optimize` 工具创建新会话并自动执行

## 执行流程

### 阶段 1：解析用户输入

1. **检测 auto 模式**：检查输入中是否包含位置性自动确认标记——紧跟在命令后出现的 `auto`、`自动`、`mode=auto`、`无需确认`、`跳过确认` 等等效表述。标记位于参数位置（紧跟 `/ae-prompt-optimize` 之后），而非嵌入在语义内容中
2. **提取实际提示词**：若检测到 auto 标记，从标记之后的内容中提取实际提示词；auto 标记本身不保留
3. **识别引用**：扫描提示词中所有符合以下格式模式的引用，记录其位置：
   - `/xxx` —— 以 `/` 开头的命令
   - `@xxx` —— 以 `@` 开头的代理
   - 包含 `:` 或 `-` 连接的多词标识符（如 `ae:brainstorm`、`task-loop`、`auto-complete`）

   匹配到的引用原样保留，不校验其是否为真实存在的命令/技能/工具/代理，不当作英语单词对待

### 阶段 2：优化提示词

对非引用部分的文本执行优化：

1. **结构化**：将模糊、口语化的表述转为结构化的任务描述（目标、约束、验收标准）
2. **补充上下文**：根据上下文推断隐含信息（如技术栈、运行环境、文件范围）并适度补充
3. **改善清晰度**：消除歧义表述，使 AI 能一次理解完整意图

**硬规则（绝不可违反）：**

- 不得改变用户原始意图和逻辑
- 首个引用必须是优化后提示词的第一个 token，前面不得添加任何文本
- 中间引用保留（可调序但不可删除）
- 引用本身原样保留，不做任何修改

**指导原则（可据情调整）：**

- 优化后提示词的长度应适中——不过度膨胀，也不遗漏关键信息
- 优先保持用户原有表述风格，仅在确实模糊时才改写
- 补充的上下文应有明确依据（用户已提及或显而易见），不凭空假设

**自检清单（优化完成后逐项确认）：**

- [ ] 用户原始意图是否完整保留？
- [ ] 首个引用是否仍在第一位？
- [ ] 所有引用是否都已保留且未被修改？
- [ ] 是否未添加任何元描述或引导文本？
- [ ] 补充的上下文是否有依据？

### 阶段 3：用户确认（auto 模式跳过）

1. **展示优化结果**：直接输出优化后的完整提示词，不得包含"这是优化后的提示词"、"优化结果如下"等任何元描述或引导文本
2. **多轮修改**：用户可反复提出修改意见，根据反馈迭代调整
   - 始终以原始输入作为参考锚点，避免多轮迭代后偏离原始意图
   - 若用户修改意见与硬规则冲突（如要求删除引用），告知冲突但遵从用户最终决定
   - 每轮修改后重新执行自检清单
3. **确认提交**：用户明确表示满意后进入阶段 4

### 阶段 4：提交执行

调用 `ae-prompt-optimize` 工具：
- `optimized_prompt`：优化后的完整提示词
- `session_title`（可选）：若用户指定了标题则传入

工具将创建新会话、发送提示词并自动导航。若工具返回错误，展示错误信息并将已确认的提示词完整展示给用户，提供重试或手动复制的选项。

## 约束规则

- 优化输出必须干净利落：不得包含"这是优化后的提示词"、"优化结果如下"等任何元描述或引导文本，直接输出提示词内容本身
- auto 标记本身不出现在优化后的提示词中
- 引用识别基于格式模式，不做语义匹配
- 不得改变用户原始意图和逻辑
