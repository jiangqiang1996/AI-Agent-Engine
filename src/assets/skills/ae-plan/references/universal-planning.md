# 通用计划工作流

本文件在 ae:plan 检测到非软件任务时加载。它替代软件特定的阶段。

## 开始之前：验证分类

- **实际上是软件任务？** 返回 SKILL.md Phase 0.2
- **快速帮助请求？** 直接回复并退出
- **管道模式？** 输出错误信息并停止——LFG 管道要求软件任务

---

## 步骤 1：评估模糊性和研究需求

**1-3 个快速问题会改善计划吗？** 默认通过步骤 1b 问 1-3 个问题。始终包含"跳过"选项。

**研究需求：**

| 需求 | 信号 | 操作 |
|------|------|------|
| **无** | 通用、永恒的计划 | 跳过研究 |
| **推荐** | 引用特定地点、价格、时间表等会过时的内容 | 在计划之前研究 |

## 步骤 1b：聚焦问答

最多 3 个针对最可能改变计划的问题。使用提问工具。提供有信息的选项而非开放式空白。

## 步骤 2：结构化计划

### 格式选择

| 任务类型 | 最佳格式 |
|----------|----------|
| 高个人偏好 | 每个类别精选选项 |
| 逻辑序列 | 单一规定路径 |
| 混合 | 带选择点的固定结构 |

### 质量原则

- 可执行的步骤、按依赖排序、时间感知、资源已识别、应急感知、适当详细

## 步骤 3：保存或分享

**问题：** "计划已就绪。你希望如何接收？"

1. **保存到磁盘** — 写为 Markdown 文件（`docs/ae/plans/` 或自定义位置）。以 `# 标题` 开始，无 YAML frontmatter。
2. **暂时完成** — 计划在聊天中呈现，不保存。

不要提供 `/ae-work` 或 Issue 创建。
