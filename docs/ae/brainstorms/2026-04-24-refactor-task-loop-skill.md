# ae:task-loop 技能重构 — 执行流程

```
/ae-task-loop "XXXXXX"
        │
        ▼
   Phase 0 — 准备（允许交互，所有用户问题在此解决）
   │  主会话仅保留：目标文本、各子会话返回的摘要、用户回答
   │
   ├─ 1. 解析用户目标（主会话）
   │
   ├─ 2. 扫描项目状态（子会话）
   │      ├─ 扫描文件/代码/配置，判断目标相关实现是否存在
   │      └─ 返回摘要：{ 是否有实现, 实现概况 }
   │
   ├─ 3. 预分析提问（子会话）
   │      ├─ 扫描目标涉及技能的 SKILL.md，提取交互节点
   │      ├─ 分析目标歧义、首次执行/验证/修复中可能需要的用户决策
   │      ├─ 可默认项（偏好类、格式类）→ 附带建议默认值
   │      └─ 返回摘要：{ 需用户回答的问题列表, 可默认项及默认值 }
   │
   ├─ 4. 收集用户回答（主会话，使用 question 工具）
   │      └─ 逐条向用户确认不可默认项
   │
   ├─ 5. 推导成功条件（子会话）
   │      ├─ 接收：目标 + 项目状态 + 用户回答
   │      └─ 返回摘要：{ 成功条件列表 [{id, description}] }
   │
   ├─ 6. 展示确认清单 + 用户确认/修改（主会话）
   │      ├─ 目标解读
   │      ├─ 当前项目状态
   │      ├─ 已预处理的交互节点及答案
   │      └─ 成功条件列表
   │      （此后全程禁止交互）
   │
   ├─ 7a. 无实现 → 首次执行（子会话）
   │        └─ 将 "xxxxx" 原文传入子会话，子会话自行解读执行
   │
   └─ 7b. 实现已存在 → 跳过首次执行
        │
        ▼
      进入循环
        │
        ▼
  循环体（子会话验证/执行，当前会话判定，全程禁止交互）
  ┌────────────────────────────────────────────────────────┐
  │  默认上限 30 轮，瓶颈阈值连续 3 轮                     │
  │                                                        │
  │  Step 1: 验证（子会话）                                │
  │  ├─ 读取当前项目状态                                   │
  │  ├─ 逐条对照成功条件：通过/不通过 + 证据              │
  │  └─ 输出结构化验证结果                                 │
  │        │                                               │
  │        ▼                                               │
  │  Step 2: 判定（当前会话）                              │
  │  ├─ 解读验证结果                                       │
  │  ├─ 记录本轮摘要（通过项、失败项）                    │
  │  │                                                     │
  │  ├─ 全部通过 ──────────────────────────→ 退出循环 ✅   │
  │  ├─ 差距较大/方向有误 → 标记 REBUILD                  │
  │  ├─ 有具体问题 → 记录问题清单，标记 FIX               │
  │  ├─ 连续 3 轮无新增通过项 ────────────→ 瓶颈退出 ⚠️  │
  │  └─ 达到 30 轮上限 ──────────────────→ 上限退出 ⚠️    │
  │        │                                               │
  │        ▼（非退出时）                                   │
  │  Step 3: 执行（子会话）                                │
  │  ├─ REBUILD：较大范围重做相关实现                      │
  │  ├─ FIX：定向修复问题清单中的问题                     │
  │  ├─ 可使用任意技能或代理                               │
  │  ├─ 遇到歧义/问题 → 自行分析选最优解，禁止提问       │
  │  └─ 返回执行结果                                       │
  │        │                                               │
  │        ▼                                               │
  │  回到 Step 1                                           │
  └────────────────────────────────────────────────────────┘
        │
        ▼
  退出处理（允许交互）
  ├─ 全部通过 → 输出 DONE
  ├─ 瓶颈退出 → 展示诊断摘要 + 选项（继续/结束/其他方案）
  └─ 上限退出 → 展示进度摘要 + 选项（增加轮次/结束/其他方案）
```
